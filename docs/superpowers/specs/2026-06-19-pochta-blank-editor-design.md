# Дизайн: Редактор бланков Почты России (переверстка на A4)

Дата: 2026-06-19
Статус: утверждён пользователем (брейншторм завершён)

## 1. Назначение

Веб-приложение на Next.js (вся обработка — в браузере). Пользователь загружает готовый PDF-бланк
с pochta.ru (таможенная декларация CN22 «Для небольших пакетов» и аналогичные) и получает
переверстанный лист A4, где:

- бланк масштабируется под размер посылки (фича «под разные размеры посылок»);
- бланк ставится на лист A4 (позиция настраивается);
- добавляются дополнительные копии трек-штрих-кода **разных размеров** для оклейки сторон посылки;
- автоматически проставляется **дата печати**;
- добавляется **поле «дата отправки»** — пустая линия для заполнения ручкой;
- результат экспортируется в готовый к печати PDF.

**Ключевое требование:** бланк должен быть **один в один** с оригиналом — все надписи и вся
разметка идентичны. Достигается переиспользованием собственной векторной графики бланка из
загруженного PDF (а не перерисовкой с нуля), что даёт побайтовую идентичность и чёткость при
любом масштабе.

## 2. Архитектура

- **Полностью клиентское** приложение, без бэкенда. Данные бланка не покидают браузер (приватность).
- Стек: **Next.js (App Router) + TypeScript + Tailwind + shadcn/ui**.
- Библиотеки обработки:
  - `pdfjs-dist` — рендер страницы для предпросмотра, чтение текстового слоя, поиск области бланка.
  - `pdf-lib` + `@pdf-lib/fontkit` — сборка и экспорт нового PDF; встраивание области бланка как
    векторного XObject (`embedPage` с обрезкой); встраивание кириллического шрифта для надписей.
  - `bwip-js` — генерация штрих-кода Code 128 (S10) для доп. копий (запасной путь / когда
    клонирование вектора недоступно).
- **Единая модель раскладки `LayoutModel`** (координаты в мм) — общий источник правды для
  предпросмотра (DOM) и для экспорта (pdf-lib). Гарантирует «что вижу, то и напечатаю».

## 3. Поток данных

1. Загрузка PDF (`File`) → открывается в pdf.js (рендер/текст) и в pdf-lib (для встраивания).
2. **Поиск области бланка**: страница 1 рендерится в canvas, по пикселям ищется непустой
   ограничивающий прямоугольник → переводится в мм. Пользователь может скорректировать рамку.
3. **Извлечение трек-номера**: из текстового слоя по шаблону S10 `[A-Z]{2}\d{9}[A-Z]{2}`
   (пример `LS018350611RU`). Не найдено → ручной ввод.
4. Формируется `LayoutModel` (позиция/ширина бланка, набор доп. штрих-кодов, флаги/позиции дат).
5. **Предпросмотр** в DOM: бланк показывается растровым изображением (только для экрана), сверху —
   HTML-слои с ручками перетаскивания/масштаба.
6. **Экспорт** (pdf-lib): новый лист A4 → `embedPage` области бланка (вектор) с масштабом и позицией →
   доп. штрих-коды (изображения от bwip-js или клон вектора) → дата печати и поле для даты →
   байты PDF → «Скачать» / «Печать».

## 4. Модули (одна ответственность на модуль)

Чистые/листовые (легко тестируются юнит-тестами):
- `lib/units.ts` — конвертация мм ↔ pt ↔ px, размеры A4.
- `pdf/extractTracking.ts` — извлечение S10-номера из текста (regex + нормализация).
- `layout/layoutModel.ts` — типы `LayoutModel`, значения по умолчанию.
- `layout/presets.ts` — пресеты посылок S/M/L → целевая ширина бланка (мм).
- `barcode/generateBarcode.ts` — обёртка над bwip-js (Code 128 → dataURL/Uint8Array).

Браузерные/интеграционные:
- `pdf/loadPdf.ts` — загрузка `File` в pdf.js и pdf-lib.
- `pdf/detectFormRegion.ts` — поиск bbox бланка по пикселям canvas.
- `render/exportPdf.ts` — сборка итогового PDF из `LayoutModel` (pdf-lib).
- `lib/fonts.ts` — встраивание кириллического TTF в pdf-lib.

Состояние и UI:
- `state/useEditorState.ts` — центральное состояние + сохранение настроек в localStorage.
- `components/UploadDropzone.tsx` — приём файла (drag&drop).
- `components/ControlsPanel.tsx` — пресеты, слайдер/мм, доп. штрих-коды, флаги дат.
- `components/PreviewCanvas.tsx` — предпросмотр A4 с перетаскиванием/масштабом.
- `app/page.tsx` — оркестрация.

## 5. Размер и положение

- Пресеты S/M/L → целевая ширина бланка в мм (стартовые ~90 / ~130 / ~180, настраиваемые).
- Тонкая настройка: слайдер масштаба + точный ввод ширины в мм + перетаскивание по листу.
- Бланк масштабируется **единым целым**, внутренняя разметка не меняется (требование «один в один»).
- Единицы в UI — мм; во внутренних расчётах для pdf-lib — pt (1 мм = 2.83465 pt; A4 = 595.28×841.89 pt).

## 6. Штрих-коды

- Собственные штрих-коды бланка — часть встроенного вектора, остаются оригинальными.
- Доп. копии: выбор размеров (S/M/L) и количества, строкой под бланком на том же листе.
- По умолчанию — клон оригинального вектора штрих-кода (обрезка `embedPage` по его bbox) и масштаб;
  запасной путь — перегенерация Code 128 по номеру (то же содержимое, читается так же).

## 7. Добавляемые элементы

- **Дата печати**: текущая дата, формат ДД.ММ.ГГГГ, подпись «Дата печати:». Вкл/выкл.
- **Поле «дата отправки»**: подпись «Дата отправки:» + пустая линия для ручки. Вкл/выкл.
- Расположение — строкой между бланком и доп. штрих-кодами; позиция настраивается.

## 8. UI / визуальное направление (shadcn)

Основа — **shadcn/ui** (Radix + Tailwind, тема через CSS-переменные). Поверх — выраженная,
намеренная эстетика, а не дефолтный вид.

**Направление: «Почтовая точность» (postal / customs precision).** Утилитарно-редакторский стиль,
вдохновлённый таможенными бланками и авиапочтой.

- **Палитра (светлая тема):** тёплый «бумажный» фон (off-white), почти чёрные чернила для текста,
  основной цвет — глубокий «почтовый синий», акцент — «штемпельный» вермильон/красный (точечно:
  активные состояния, важные действия). Тёмная тема — опционально, позже.
- **Типографика (не Inter/Roboto/Space Grotesk):** заголовки — индустриальный гротеск **Archivo**
  (с расширенным трекингом для крупных заголовков); основной текст — **Hanken Grotesk**;
  моноширинный — **JetBrains Mono** для трек-номеров, размеров (мм), процентов масштаба. Подключение
  через `next/font/google`.
- **Фирменные детали:** пунктирные «перфорированные»/авиапочтовые рамки (сине-красные), едва заметная
  бумажная зернистость, регистрационные/обрезные метки по углам панелей, моноширинные «штемпельные»
  индикаторы значений, мотив штрих-кода в шапке.
- **Движение (сдержанно):** ступенчатое появление панелей при загрузке, плавные перетаскивание/масштаб,
  лёгкий подъём контролов при наведении.
- **Компоненты shadcn:** button, card, slider, input, label, switch, tabs, separator,
  tooltip, sonner (тосты), dropdown-menu.

## 9. Обработка ошибок

- Не-PDF/битый файл → понятное сообщение, повторная загрузка.
- Трек-номер не найден → поле ручного ввода, не блокирует.
- Область бланка не распозналась → запасная рамка (верх страницы) + ручная корректировка.
- Многостраничный PDF → берётся страница 1 (с возможностью выбрать другую).
- Сбой генерации штрих-кода/экспорта → предупреждение (тост), приложение не падает.
- Отсутствие кириллических глифов → гарантируется встраиванием TTF; покрывается тестом.

## 10. Тестирование

- **Юнит (TDD на чистых функциях):** `units` (конвертации), `extractTracking` (regex на тексте
  образца), `presets`/`layoutModel` (математика масштаба и умолчаний).
- **Интеграция:** образец PDF (`samples/sample-blank.pdf`) как фикстура — извлечение номера из
  текста; сборка PDF (где запускаемо headless) — один лист A4, ожидаемые объекты/изображения.
- **Визуальная сверка:** сравнение экспортированного бланка с оригиналом (фиделити) вручную.

## 11. Стек и запуск

Next.js (App Router) + TS + Tailwind + shadcn/ui; `pdfjs-dist`, `pdf-lib`, `@pdf-lib/fontkit`,
`bwip-js`; кириллический TTF; vitest для тестов. Локально `npm run dev`; деплой — Vercel.

## 12. Вне рамок (YAGNI)

Аккаунты/авторизация, серверное хранение, адресная книга, редактирование внутренних полей бланка,
пакетная обработка, форматы листа кроме A4 — сейчас не делаем.

## 13. Главные риски

- Точность автоопределения рамки бланка/штрих-кода — митигируется ручной корректировкой.
- Встраивание кириллического шрифта в pdf-lib (через fontkit) — покрывается тестом.
- Размер бандла pdf.js+pdf-lib в браузере — допустимо для инструмента.
- Расхождение предпросмотра и экспорта — митигируется единой `LayoutModel` (мм).

## 14. Последовательность сборки (для реализации субагентами)

1. Каркас: create-next-app + Tailwind + shadcn init + установка библиотек + перенос образца PDF в
   `samples/`. Тема и шрифты (globals.css, next/font).
2. Общие контракты: `lib/units.ts`, `layout/layoutModel.ts` (типы) — стабильные интерфейсы для всех.
3. Параллельно (субагенты, по отдельным файлам + тесты): `units`, `extractTracking`, `presets`,
   `generateBarcode`, `detectFormRegion`, `loadPdf`, `exportPdf`.
4. UI/эстетика: `UploadDropzone`, `ControlsPanel`, `PreviewCanvas`, `app/page.tsx`, тема shadcn.
5. Интеграция, проверка типов и сборки, запуск dev, визуальная сверка.
